## A estrutura

```
projeto/
├── CLAUDE.md          ← memória e regras do projeto
├── DESIGN.md          ← identidade visual (ver 6.4)
├── docs/
│   ├── decisions/     ← por que cada decisão foi tomada
│   └── runbooks/      ← guias de operação
├── .claude/
│   ├── hooks/         ← verificações automáticas
│   └── skills/        ← workflows reutilizáveis
│       ├── code-review/SKILL.md
│       ├── refactor/SKILL.md
│       └── release/SKILL.md
└── src/
    └── api/CLAUDE.md  ← instruções específicas do módulo
```

## Os arquivos raiz

Pense no **CLAUDE.md** como um caderno de regras que a IA lê toda vez que começa a trabalhar no seu projeto. É como deixar um bilhete para um funcionário novo explicando como as coisas funcionam aqui. O **DESIGN.md** é como um guia de estilo que diz quais cores e letras usar — a IA aplica isso automaticamente em tudo que criar.

## O diretório `.claude/`

Os **hooks** (verificações automáticas) funcionam como um revisor que checa seu trabalho automaticamente antes de enviar — igual a um corretor ortográfico que roda sozinho. As **skills** (habilidades) são como receitas prontas que a IA pode seguir, por exemplo: "siga esses passos para revisar um código".

## CLAUDE.md por módulo

Cada parte do projeto pode ter seu próprio caderno de regras. É como ter um manual diferente para a cozinha e outro para o salão de um restaurante — cada área tem suas próprias instruções específicas.

## O loop de melhoria

```
Claude erra → você adiciona regra ao CLAUDE.md
Você se repete → você cria uma skill
Algo quebra → você adiciona um hook
= Claude fica mais inteligente continuamente
```

A cada erro que a IA comete, você escreve uma regra para evitar que aconteça de novo. Com o tempo, é como treinar um funcionário — ele vai errando cada vez menos porque as instruções ficam mais completas.

## Por que isso muda tudo

Antes era como contratar alguém novo para cada projeto, precisando explicar tudo do zero toda vez. Agora você guarda o conhecimento do projeto e a IA fica cada vez mais afinada com o jeito que você trabalha. O esforço é maior no começo, mas depois o trabalho flui muito mais fácil.

## Exemplos do dia a dia

1. Uma pequena padaria usa esse modelo para guardar as regras do negócio: "sempre usa farinha integral", "o horário de entrega é sempre de manhã". Cada vez que pedem ajuda à IA para criar um novo cardápio, ela já sabe essas regras sem precisar perguntar de novo.

2. Um fotógrafo freelancer cria regras como "minhas fotos sempre têm fundo branco" e "o logo vai sempre no canto inferior direito". A IA aplica essas preferências automaticamente em qualquer material novo que criar — sem precisar lembrar a ela toda vez.
