## O que é o Claude Code e por que ele precisa de um método

Claude Code é uma ferramenta de inteligência artificial criada pela Anthropic — a mesma empresa que faz o Claude, aquele assistente de IA que muita gente já conhece pelo chat. A diferença é que o Claude Code é feito especialmente para ajudar a criar programas, sites e sistemas. Mas, assim como qualquer ferramenta poderosa, ele tem muito mais capacidade do que a maioria das pessoas usa.

Pense numa cozinha profissional de restaurante. Ela tem fogão industrial, fornos combinados, processadores de alta velocidade, câmaras frias, bancadas organizadas e toda uma estrutura pensada para produzir refeições em grande quantidade e com alta qualidade. Agora imagine usar essa cozinha profissional só para fazer ovo mexido, sem usar nenhum dos outros equipamentos. É exatamente assim que a maioria das pessoas usa o Claude Code — pela fração mínima do que ele pode fazer.

Os 8 conceitos que vamos ver a seguir são os "equipamentos" dessa cozinha profissional. Entender cada um deles é o que separa quem usa a IA como uma calculadora de quem a usa como uma equipe inteira de trabalho.

## Por que isso importa para você?

Se você usa o Claude Code só para fazer perguntas e receber respostas, está perdendo a maior parte do valor da ferramenta. É como alugar um carro esportivo e andar sempre em segunda marcha. Os 8 conceitos abaixo permitem que você construa um sistema onde a IA trabalha por você de forma automática, consistente e cada vez mais inteligente — sem precisar repetir as mesmas instruções toda vez.

O impacto no dia a dia é enorme: tarefas que você fazia manualmente passam a ser automáticas. Erros que você cometia por esquecimento passam a ser capturados automaticamente. Processos que levavam horas passam a levar minutos. A IA vai de "assistente que responde perguntas" para "membro da equipe que trabalha enquanto você dorme".

Quem domina esses 8 conceitos consegue produzir em uma semana o que levaria um mês no modo tradicional. Não porque trabalha mais — mas porque configurou a IA para trabalhar melhor.

## Os 8 conceitos fundamentais explicados

| Conceito | O que faz | Como entender |
|---|---|---|
| **Commands** | Atalhos rápidos para tarefas inteiras | Como um botão na padaria que, quando apertado, gera o pedido completo do dia automaticamente |
| **Sub-Agents** | Assistentes menores especializados dentro da IA principal | Como ter um auxiliar de cozinha que cuida só das sobremesas enquanto o chef principal faz o prato principal |
| **Skills** | Receitas prontas que a IA reutiliza sempre que precisar | Como um livro de receitas guardado na gaveta — consulta quando precisa, sem inventar do zero |
| **Workflows** | Sequências automáticas de tarefas que acontecem uma após a outra | Como a linha de montagem de uma fábrica — cada etapa aciona a próxima, sem precisar de ninguém no meio |
| **Hooks** | Gatilhos automáticos que disparam ações quando algo acontece | Como o sensor da porta que acende a luz quando você entra — aconteceu o evento, a ação veio junto |
| **MCP Servers** | Conexões com outros serviços externos | Como o telefone que liga a cozinha ao fornecedor — quando precisa de algo de fora, a linha já existe |
| **Plugins** | Pacotes completos que juntam vários dos itens acima num só lugar | Como um kit de cozinha completo: fogão, panelas e utensílios num único pacote |
| **Settings** | As regras gerais de como toda a operação funciona | Como o regulamento interno da cozinha — o que pode, o que não pode, como as coisas devem ser feitas |

## Entendendo cada conceito com mais profundidade

### Commands — os atalhos poderosos

Um Command é como criar um botão personalizado que faz uma tarefa inteira com um único clique. Pense assim: um funcionário de RH toda semana precisa gerar um relatório de horas trabalhadas, calcular os descontos, formatar num documento e enviar por email. São 4 etapas repetitivas. Um Command faria tudo isso com uma única palavra digitada — como apertar um botão e ver a máquina fazer o trabalho.

No Claude Code, você cria um Command escrevendo uma vez as instruções de como aquela tarefa deve ser feita. Daí em diante, é só chamar o Command pelo nome e a IA executa tudo, na ordem certa, sem precisar de mais explicações. Você para de repetir as mesmas instruções e começa a só acionar o gatilho.

### Sub-Agents — a equipe dentro da equipe

Um Sub-Agent é como ter um especialista dentro do seu time que cuida só de uma área específica. Imagine um hospital: o médico geral coordena o caso, mas chama o cardiologista para o coração, o neurologista para o cérebro, o ortopedista para os ossos. Cada um é especialista no que faz. A IA principal age como o médico geral, e os Sub-Agents são os especialistas.

Isso importa porque uma IA que tenta fazer tudo ao mesmo tempo pode se perder. Quando você divide o trabalho em especialistas menores, cada parte é feita melhor — como montar um time onde cada pessoa tem uma função clara.

### Skills — o conhecimento que fica guardado

Uma Skill é um conhecimento que você ensina uma vez e a IA guarda para usar sempre que precisar. Diferente de uma pergunta que você faz e recebe uma resposta e pronto — uma Skill fica disponível permanentemente, como um funcionário que aprendeu como você gosta das coisas e nunca mais precisa perguntar de novo.

### Workflows — o piloto automático

Um Workflow é uma sequência de etapas que acontece automaticamente. Pense numa lavanderia: roupa suja entra, passa pela lavagem, pela centrifugação, pela secagem, é dobrada e sai pronta do outro lado. Ninguém precisa carregar a roupa de uma etapa para outra manualmente — é automático. Com Workflows, você monta essa "linha de produção" para a sua IA, e os processos fluem sozinhos de uma etapa para outra.

### Hooks — os guardas automáticos

Hooks são como guardas de segurança que ficam de olho o tempo todo e agem automaticamente quando algo específico acontece. É como o alarme do carro: você não precisa ficar olhando para o carro — se alguém tocar, o alarme dispara sozinho. No Claude Code, você pode criar um Hook que, por exemplo, toda vez que a IA terminar de escrever um código, automaticamente verifica se tem erros antes de finalizar — como um revisor que entra em cena automaticamente quando o escritor termina o rascunho.

### MCP Servers — as pontes para o mundo

MCP Servers são conexões que permitem que a IA fale com outros programas e serviços. É como a tomada universal que conecta diferentes aparelhos à energia elétrica — mas aqui, conecta a IA ao GitHub (onde ficam os códigos), ao Google Drive (onde ficam os documentos), ao Slack (onde ficam as mensagens da equipe) e muito mais. Sem MCP, a IA fica isolada. Com MCP, ela consegue buscar informações, salvar resultados e acionar ações em outros sistemas, como uma secretária que atende vários telefones ao mesmo tempo.

## Como esses conceitos se encaixam entre si

Tudo funciona em camadas, de baixo para cima. No nível mais básico estão as Settings — as regras gerais. Em cima delas, os MCP Servers trazem informações e conexões do mundo externo. Sobre isso, as Skills guardam o conhecimento especializado. Os Commands juntam tudo em atalhos rápidos. Os Hooks ficam de vigia, disparando ações quando necessário. Os Workflows coordenam sequências inteiras. Os Sub-Agents se especializam em partes específicas. E os Plugins empacotam tudo isso num conjunto pronto para instalar.

É como montar um restaurante do zero: primeiro você define as regras da casa, depois conecta com fornecedores, contrata especialistas, cria o cardápio, treina os funcionários e monta os processos. Cada camada faz a próxima funcionar melhor.

## O caminho recomendado — do simples ao complexo

Não tente aprender tudo de uma vez. Isso é receita de frustração. O caminho que funciona é:

1. Use o chat básico por pelo menos uma semana — conheça a ferramenta no modo mais simples
2. Crie seu primeiro Command para a tarefa que você faz com mais frequência
3. Crie uma Skill para o tipo de trabalho que você mais repete no seu dia a dia
4. Conecte um MCP Server com um serviço que você já usa, como o GitHub
5. Só depois explore Sub-Agents e Workflows para automatizar processos mais complexos
6. Quando tudo isso estiver funcionando, monte um Plugin que junte tudo num pacote reutilizável

Esse caminho leva de algumas semanas a poucos meses. Mas cada etapa já entrega valor imediato — você não precisa terminar tudo para começar a se beneficiar.

## Exemplos do dia a dia

1. Um dono de padaria quer que a IA ajude a administrar o negócio. Ele configura um Command chamado "semana" que, quando acionado, automaticamente gera o cardápio da semana baseado nos ingredientes disponíveis, calcula a quantidade necessária de cada item, cria uma lista de compras organizada por fornecedor e gera o custo estimado. Antes, ele fazia isso manualmente em 2 horas toda segunda-feira. Com o Command, leva 3 minutos — e o resultado é mais organizado e completo do que fazia sozinho.

2. Um professor de escola particular cria uma Skill que ensina à IA como é o seu estilo de corrigir redações: os critérios que usa, o tipo de feedback que gosta de dar, o nível de detalhe que considera ideal para cada faixa etária. Depois de criar essa Skill uma única vez, toda vez que precisar corrigir uma redação, a IA já sabe exatamente como ele gosta — o mesmo jeito, com a mesma qualidade, sem precisar explicar os critérios de novo. Ele passa de 30 minutos por redação para 5 minutos, sem perder a qualidade do feedback.

3. Uma pequena empresa de contabilidade configura um Workflow completo para processar documentos de clientes: quando um cliente envia seus documentos por email, um Hook captura o email automaticamente, um Sub-Agent especializado em documentos fiscais os organiza e classifica, outro Sub-Agent verifica se está tudo completo, e um terceiro gera o rascunho da declaração. O contador recebe o rascunho pronto para revisar e assinar — em vez de passar o dia organizando papelada, ele passa o dia revisando e atendendo clientes.
