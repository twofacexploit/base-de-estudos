## Os 8 conceitos fundamentais

| Conceito | O que é |
|---|---|
| **Commands** | Atalhos que executam tarefas com uma palavra (`/revisar-codigo`) |
| **Sub-Agents** | IAs menores e especializadas trabalhando dentro do Claude |
| **Skills** | Blocos de conhecimento reutilizáveis para tarefas específicas |
| **Workflows** | Sequências automáticas de tarefas encadeadas |
| **Hooks** | Scripts que disparam automaticamente em eventos (commit, save...) |
| **MCP Servers** | Conexões com ferramentas e serviços externos |
| **Plugins** | Pacotes que combinam Skills + Agents + Hooks + MCPs |
| **Settings** | Permissões, sandboxing, output styles, fast mode |

## Como esses conceitos se encaixam

Pense em camadas:

```
Settings        → define regras globais
   ↓
MCP Servers     → conectam a mundos externos
   ↓
Sub-Agents      → especialistas dedicados
   ↓
Skills          → conhecimento reutilizável
   ↓
Workflows       → sequências automatizadas
   ↓
Commands/Hooks  → atalhos e gatilhos
```

## Por que isso importa

A maior parte dos iniciantes usa Claude Code como **chat glorificado** — só pergunta e recebe. Quem domina esses 8 conceitos transforma o Claude numa **plataforma**, não numa ferramenta. A diferença de produtividade chega a 10x.

## O caminho recomendado

1. Aprenda a usar o chat básico (1 semana)
2. Escreva seu primeiro **Command** pessoal
3. Crie uma **Skill** para sua stack principal
4. Conecte um **MCP Server** (ex: GitHub, Postgres)
5. Só então, tente **Sub-Agents** e **Workflows** complexos
