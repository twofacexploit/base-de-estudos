## URL

`insforge.dev`

## O diferencial

Pense no **backend** (a parte de trás de um site ou aplicativo) como a cozinha de um restaurante — é onde tudo é preparado antes de chegar ao cliente. O **Supabase** é uma cozinha excelente, mas projetada para cozinheiros humanos. O **Insforge** é uma cozinha projetada para robôs cozinheiros — os agentes de IA já sabem onde fica cada ingrediente, sem precisar que alguém explique.

## O que oferece

### Banco de dados
O mesmo tipo de banco de dados confiável que o Supabase usa — como uma planilha gigante e organizada que guarda todas as informações do seu aplicativo.

### Sistema de login
Permite que usuários entrem no seu app com e-mail, redes sociais ou link mágico enviado por e-mail.

### IA nativa
- **Busca inteligente** embutida — encontra coisas pelo significado, não só pela palavra exata
- **Perguntas e respostas com documentos** prontos — você manda um documento, o usuário faz perguntas, o sistema responde com base no conteúdo

## O que pode substituir

Em projetos que usam muito IA, o Insforge pode substituir duas ferramentas de uma vez:

```
Supabase + Pinecone     →     Insforge
(banco + login)            (banco + login + busca inteligente)
  + $25/mês                   tudo integrado
```

## Habilidades integradas para agentes de IA

O ponto mais especial: quando você usa o Claude (o assistente de IA) com o Insforge, ele já sabe como usar o sistema sozinho. O Claude já consegue:

- Criar tabelas no banco de dados
- Organizar e indexar informações automaticamente
- Fazer buscas inteligentes
- Responder perguntas usando documentos

Isso economiza horas de configuração em cada projeto novo.

## Quando escolher Insforge vs Supabase

| Caso | Escolha |
|---|---|
| Projeto tradicional, pessoas escrevendo o código | **Supabase** (mais maduro e testado) |
| Projeto onde a IA faz grande parte do trabalho | **Insforge** (menos dor de cabeça) |
| Quando busca inteligente é central no produto | **Insforge** (já vem incluído) |
| Time grande que precisa de estabilidade corporativa | **Supabase** (ecossistema maior) |

## A tendência

Ferramentas novas estão sendo construídas pensando que quem vai usá-las é uma IA, não uma pessoa. Isso muda tudo: a forma como o sistema responde a erros, como organiza a documentação, como nomeia as funções. É uma mudança parecida com quando os sites deixaram de ser feitos só para computadores e passaram a ser feitos também para celulares.

## Exemplos do dia a dia

1. **App de perguntas e respostas sobre documentos:** uma empresa de RH quer um sistema onde funcionários fazem perguntas sobre o manual da empresa e recebem respostas automáticas. Com o Insforge, você sobe o manual e o sistema já responde às perguntas — sem precisar configurar nada extra.

2. **Assistente de atendimento ao cliente:** uma loja online quer um chatbot que responde sobre produtos, pedidos e políticas de devolução. Com o Insforge, o agente de IA já sabe como acessar o banco de dados dos pedidos e encontrar as informações certas para cada cliente.
