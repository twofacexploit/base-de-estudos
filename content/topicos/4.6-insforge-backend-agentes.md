## URL

`insforge.dev`

## O diferencial

O **Supabase** é excelente, mas foi desenhado para ser usado por **humanos** escrevendo código. O **Insforge** foi construído desde o primeiro dia para ser usado por **agentes de IA** — com skills integradas para que o Claude, Codex, Cursor já saibam como usar o banco sem configuração extra.

## O que oferece

### PostgreSQL
Mesmo banco que Supabase usa — confiável, escalável, padrão do mercado.

### Autenticação
Sistema completo de auth (email/senha, OAuth social, magic links).

### IA nativa
- **Vector embeddings** embutidos (não precisa plugar Pinecone)
- **Busca semântica** pronta
- **RAG built-in** — você manda documentos, faz perguntas, ele responde com contexto

## O que pode substituir

Em projetos **centrados em agentes**, o Insforge pode substituir duas ferramentas do stack tradicional:

```
Supabase + Pinecone     →     Insforge
(banco + auth)            (banco + auth + vector db + RAG)
  + $25/mês                   tudo integrado
```

## Agent Skills integradas

O ponto-chave: quando você usa Claude Code com Insforge, as skills **já estão instaladas**. O Claude já sabe como:

- Criar tabelas no formato esperado
- Popular embeddings automaticamente
- Fazer queries vetoriais
- Implementar RAG sem você ter que ensinar

Economiza **horas** de configuração por projeto.

## Quando escolher Insforge vs Supabase

| Caso | Escolha |
|---|---|
| Projeto tradicional, humanos codando | **Supabase** (mais maduro) |
| Projeto IA-first, agentes dominantes | **Insforge** (menos fricção) |
| Busca semântica central | **Insforge** (vector db incluído) |
| Time grande, precisa de estabilidade enterprise | **Supabase** (ecossistema maior) |

## A tendência

Novas ferramentas estão sendo construídas **"AI-first"** — assumindo que quem vai usá-las é um agente, não um humano. Isso é uma mudança arquitetural importante: o **consumidor do API** agora é uma IA, e isso afeta design de tudo: documentação, erros, APIs, nomenclatura.
