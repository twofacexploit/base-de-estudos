## O que é XSS

**XSS = Cross-Site Scripting.** Injetar JavaScript malicioso num site — quando outra pessoa visita, o script roda no browser dela. Consequências: roubo de sessão, redirect pra phishing, keylogger invisível.

## Os 4 tipos

```
Reflected:  URL maliciosa → executa só para quem clica no link
Stored:     Salvo no banco → executa para TODOS os visitantes
DOM-based:  Manipula o DOM sem passar pelo servidor
Blind XSS:  Executa em página admin que você não vê (mas monitora)
```

**Stored XSS** é o pesadelo: você injeta uma vez num comentário de blog, e milhares de visitantes executam o código sem saber.

## Bypass de filtros — exemplos

```html
Básico (bloqueado pela maioria dos WAFs):
<script>alert(1)</script>

Com evento HTML:
<img src=x onerror=alert(1)>

Com SVG (frequentemente ignorado):
<svg onload=alert(1)>

Com encoding para quebrar o parser:
<scr\x00ipt>alert(1)</scr\x00ipt>
```

A arte do XSS moderno é **descobrir o filtro** e criar um payload que passa por ele. Cada WAF tem seus pontos cegos.

## Defesa básica

- **Output encoding** — escapar `<`, `>`, `"`, `'` antes de imprimir no HTML
- **Content Security Policy (CSP)** — bloqueia scripts inline por padrão
- **Frameworks modernos** (React, Vue, Angular) — fazem escape automático
