## O autor

**Souhaib Naceri** (conhecido online como h4x0r_dz) — pesquisador de segurança argelino famoso por pegar problemas pequenos e combiná-los para criar impactos gigantes.

## A lição principal

Problemas isolados costumam pagar pouco. Quando você encontra 3 problemas pequenos que se conectam — onde um abre a porta para o próximo — o impacto cresce muito e o pagamento também. Este case é um exemplo perfeito disso.

## As 3 vulnerabilidades encadeadas

### 1. Manipulação de "crachá" de acesso — $3.000

Imagine um sistema onde cada pessoa tem um crachá com seu cargo escrito. Naceri descobriu que podia escrever qualquer cargo nesse crachá, e o sistema aceitava sem questionar. Ele escreveu "administrador" e ganhou acesso ao painel de administração.

```
Login normal:  {"realm": "test-user"}
Ataque:        {"realm": "test-dashboard"}
Resultado:     Acesso ao painel admin
```

### 2. Porta dos fundos no sistema de autenticação — $20.000 (compartilhado)

É como se a porta principal de um banco verificasse a senha, mas descobrisse que a porta dos fundos deixava qualquer um entrar se removesse uma palavra específica do crachá.

```
Com identificador:    Authorization: Bearer TOKEN → Bloqueado
Sem identificador:    Authorization: TOKEN        → Liberado
```

### 3. Upload de arquivo sem controle de destino

Naceri descobriu que ao enviar um arquivo, ele podia escolher para onde esse arquivo ia dentro dos servidores da empresa — sem nenhum controle. Poderia substituir:

- O JavaScript do site principal (controlando o que todos os usuários veem)
- Arquivos de instalação que usuários baixam
- Documentos oficiais publicados no site

Resultado teórico: ele poderia colocar um vírus num arquivo que milhares de usuários baixariam sem saber.

## Ferramentas usadas

- **Burp Suite** — interceptar e modificar as comunicações com o site
- **ffuf** — testar centenas de endereços em busca de páginas escondidas
- **Wayback Machine** — encontrar páginas antigas removidas do site mas ainda acessíveis
- **JWT.io** — ler e modificar os "crachás" digitais de acesso

## A plataforma

Relatório enviado pelo **HackerOne**. Pagamento total recebido: **$23.000**.

## A moral

Nenhum desses três problemas sozinho seria considerado grave. Mas combinados, formaram um cenário de desastre potencial — e o pesquisador foi recompensado de acordo com o impacto real que poderia causar.

## Exemplos do dia a dia

1. É como descobrir que a catraca de uma balada não verifica o crachá corretamente (problema 1), que a porta dos fundos fica destrancada (problema 2), e que dentro da balada dá pra trocar o sistema de som sem que ninguém perceba (problema 3). Cada coisa sozinha é menor; juntas, você controla a noite toda.

2. Um pesquisador iniciante encontra apenas o primeiro problema e recebe $500. Naceri encontrou os três e mostrou como um preparava o caminho para o outro — recebendo 46 vezes mais. A habilidade de encadear falhas é o que separa um bom profissional de um especialista de alto valor.
