## O prompt completo

```
You'll act as a professional code reviewer and tech lead 
for this whole codebase. Review all my project code structure, 
each file, each page, each endpoint, each logic, and write me 
a QUESTIONS.md with all the architectural, refactoring and 
technical questions you found odd or need more info about.
[...] ultrathink about this
```

## O comando mágico: "ultrathink"

**"ultrathink"** força o raciocínio máximo do Claude — ele pensa por muito mais tempo antes de responder, explorando mais hipóteses e detalhes. É a diferença entre uma revisão rápida e uma **auditoria profunda**.

## Resultado real (exemplo documentado)

Em um único run, gerou **68 perguntas** organizadas em **7 categorias**:

| Categoria | Qtd |
|---|---|
| Security | 17 |
| Architecture | 27 |
| Frontend | 16 |
| Data Model | 10 |
| Testing | 4 |
| Code Quality | 7 |
| Performance | 5 |

Tudo escrito em `QUESTIONS.md` — um documento que normalmente levaria um tech lead humano dias pra produzir.

## O fluxo que funciona

```
Prompt → QUESTIONS.md gerado → você responde as perguntas
       → Claude implementa as correções → iteração
```

Você responde pergunta por pergunta, e a IA implementa as decisões. É code review **colaborativo** — humano decide o quê, máquina faz o como.

## Por que isso importa

Revisão de código profunda é cara: um tech lead sênior custa hora de R$200+ e faz 20-40 horas de audit num projeto médio. Esse prompt entrega **80% do valor em minutos**, praticamente de graça (só o custo dos tokens).
