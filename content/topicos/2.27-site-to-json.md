## O que resolve

**"Design com IA é difícil de padronizar"** — esse é o problema real. Se você pede pro Claude construir uma página "parecida com o Stripe", ele interpreta do jeito dele. Na próxima página, diferente. Sem consistência.

A solução: **extrair o design em JSON estruturado** que a LLM **sempre** entende igual.

## URL

`site-to-json.trapiche.cloud`

## O fluxo

```
URL do site de referência  
   ↓  
Site-to-JSON extrai:  
   - Cores (primária, secundária, texto, background)  
   - Tipografia (famílias, tamanhos, pesos)  
   - Espaçamentos (ritmo, padding, margin)  
   - Componentes (botões, cards, forms)  
   ↓  
JSON estruturado  
   ↓  
Alimenta o agente de IA  
   ↓  
Resultado consistente SEMPRE  
```

## O JSON como "contrato"

JSON é uma linguagem que **LLMs interpretam de forma confiável**. Diferente de descrição em texto (ambígua), JSON é exato. Se você diz:

```json
{
  "colors": {
    "primary": "#635BFF",
    "bg": "#0A2540"
  }
}
```

A IA **nunca** vai inventar outra cor. Consistência garantida.

## Integra com

- **Claude Code**
- **Cursor**
- Qualquer agente que lê contexto estruturado
- **DESIGN.md** (6.4) — você pode transformar o JSON em um DESIGN.md completo

## Por que isso é um salto

Antes: você explicava o design em palavras e torcia pra IA entender. Agora: você **mostra** o design e a IA **aplica**. Muda o modelo de trabalho de *"prompt engineering"* pra *"context engineering"* — que é onde o futuro está.
