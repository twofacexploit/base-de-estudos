## O que resolve

Quando você pede para uma IA criar uma página "parecida com o Stripe" (site de pagamentos famoso pelo visual elegante), ela interpreta do jeito dela. Na próxima página, interpreta diferente. Nada fica consistente. A solução é **extrair o design real do site** e entregar para a IA em um formato que ela sempre entende igual — como dar uma receita exata em vez de falar "faça algo gostoso".

## URL

`site-to-json.trapiche.cloud`

## O fluxo

```
URL do site de referência  
   ↓  
Site-to-JSON extrai:  
   - Cores (primária, secundária, texto, fundo)  
   - Tipografia (tipo de letra, tamanhos, espessura)  
   - Espaçamentos (distâncias entre elementos)  
   - Componentes (botões, cartões, formulários)  
   ↓  
JSON estruturado  
   ↓  
Alimenta o agente de IA  
   ↓  
Resultado consistente SEMPRE  
```

## O JSON como "contrato"

JSON é um formato de dados muito preciso que as IAs entendem perfeitamente. É como uma receita de bolo com medidas exatas — sem margem para improviso. Se você diz:

```json
{
  "colors": {
    "primary": "#635BFF",
    "bg": "#0A2540"
  }
}
```

A IA **nunca** vai inventar outra cor. Consistência garantida.

## Por que isso é um salto

Antes: você explicava o design em palavras e torcia para a IA entender. Agora: você **mostra** o design e a IA **aplica exatamente**. É a diferença entre falar "pintura na cor do oceano" e mostrar a tinta com o código de cor exato — não tem como errar.

## Exemplos do dia a dia

1. Uma loja online quer que a IA crie todas as páginas no mesmo estilo visual de uma loja famosa que ela admira. Em vez de tentar explicar "quero aquele tom de azul e aquela fonte", ela cola o link do site de referência no Site-to-JSON, pega o resultado e entrega para a IA — que cria tudo com perfeita fidelidade visual.

2. Um designer que trabalha para vários clientes quer que a IA mantenha a identidade visual de cada marca. Ele extrai o JSON do site de cada cliente e usa como guia — a IA nunca mistura as cores do cliente A com as do cliente B.
