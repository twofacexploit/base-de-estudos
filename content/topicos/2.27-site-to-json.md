## O que é isso, em linguagem simples

Imagine que você quer que um pasteleiro recrie a sobremesa favorita de outro restaurante. Você pode descrever para ele: "era uma torta, mais ou menos escura, com uma cobertura brilhante, não muito doce". Mas sem a receita exata — as medidas, os ingredientes precisos, o tempo de forno —, o resultado vai ser parecido mas nunca igual. Cada vez que você pede, sai diferente. Depende do humor e da interpretação do pasteleiro.

Agora imagine que você tem a receita completa com medidas precisas: "200g de chocolate 70%, 150g de manteiga, temperatura de 180 graus por 22 minutos". Agora sim, qualquer pasteleiro consegue recriar exatamente a mesma torta, toda vez, sem variação.

O **Site-to-JSON** faz isso com design de sites. Você cola o endereço de um site que admira, e a ferramenta extrai a "receita" visual exata daquele site: as cores precisas (não "azul escuro" — o código hexadecimal exato que define aquele azul), o tipo e tamanho das letras, os espaçamentos entre elementos, os estilos dos botões. Tudo em um formato que a IA entende perfeitamente e reproduz com fidelidade absoluta.

## Por que isso importa para você?

O problema mais comum ao usar IA para criar materiais visuais — sites, posts, apresentações — é a inconsistência. Você pede um site "no estilo do Stripe" (plataforma de pagamentos famosa pelo design elegante) e a IA cria algo parecido. Você pede outra página "no mesmo estilo" e ela cria algo diferente. Você nunca consegue garantir que dois materiais pareçam que vieram da mesma marca.

Isso acontece porque palavras são ambíguas. "Azul elegante" pode ser interpretado de mil formas. "Espaçamento generoso" é diferente para cada pessoa. Mas um código de cor é absoluto — `#635BFF` é exatamente aquele roxo específico do Stripe, sempre, em qualquer lugar do mundo.

O Site-to-JSON transforma a descrição imprecisa em dados precisos. A IA deixa de adivinhar e passa a seguir uma receita exata. O resultado: consistência garantida em todos os materiais que você criar.

## O endereço da ferramenta

`site-to-json.trapiche.cloud`

Você acessa pelo navegador, cola o endereço do site de referência, e recebe o resultado.

## Como funciona — o fluxo completo

```
Você cola o endereço do site que admira
         ↓
A ferramenta analisa o site inteiro
         ↓
Extrai os elementos de design:
   - Cores (primária, secundária, texto, fundo, links)
   - Tipografia (tipo de letra, tamanhos, espessura, espaçamento entre linhas)
   - Espaçamentos (distâncias entre elementos na página)
   - Componentes (como são os botões, cartões, formulários, menus)
         ↓
Entrega tudo em formato JSON
         ↓
Você alimenta esse JSON para a IA
         ↓
A IA cria materiais sempre no mesmo estilo visual
```

## O que é JSON — explicado sem jargão

**JSON** é simplesmente uma forma de organizar informações que o computador entende com precisão absoluta. Pense nele como uma ficha técnica super detalhada. Quando você escreve numa ficha que a cor primária é `#635BFF`, não tem margem para interpretação. Não é "um roxo bonito" — é aquele roxo específico.

Veja um exemplo do que a ferramenta gera:

```json
{
  "colors": {
    "primary": "#635BFF",
    "background": "#0A2540",
    "text": "#FFFFFF",
    "accent": "#80E9FF"
  },
  "typography": {
    "font": "Helvetica Neue",
    "heading_size": "48px",
    "body_size": "18px"
  }
}
```

Com esse arquivo na mão, a IA sabe exatamente quais cores usar, qual fonte, qual tamanho. Não inventa nada. Não improvisa. Segue a receita.

## Por que isso representa um salto de qualidade

Antes do Site-to-JSON, o processo era assim: você explicava em palavras como queria o design, a IA interpretava do jeito dela, você olhava o resultado e dizia "não é bem assim", ela tentava de novo com uma interpretação diferente, você tentava descrever melhor... Era um jogo de adivinhação frustrante.

Com o Site-to-JSON, o processo é: você mostra o que quer, a ferramenta extrai os dados exatos, a IA aplica exatamente. Não tem campo para interpretação. É a diferença entre mostrar a tinta com o código de cor exato versus tentar descrever "um azul parecido com o oceano no início da tarde num dia ensolarado".

## Exemplos do dia a dia

1. Uma microempreendedora chamada Cláudia tem uma marca de cosméticos naturais e admira profundamente o design do site de uma marca internacional do setor. Ela não tem dinheiro para contratar um designer que cobrou R$8.000 para criar o site. Com o Site-to-JSON, ela extrai os dados visuais do site que admira — as cores neutras, a tipografia elegante, o espaçamento generoso — e entrega esse arquivo para a IA junto com os produtos dela. A IA cria um site com aquele mesmo nível visual sofisticado, só que com as cores da marca dela, as fotos dos produtos dela, o texto dela. O resultado parece profissional porque segue uma receita profissional real.

2. Um designer freelancer chamado Bruno trabalha para cinco clientes diferentes ao mesmo tempo. O maior desafio é não misturar a identidade visual de um cliente com a de outro. Quando pede para a IA criar materiais para o cliente A, às vezes ela "se lembra" de cores do cliente B. Com o Site-to-JSON, Bruno extrai um JSON para cada cliente e usa como referência obrigatória quando trabalha para aquela marca. A IA nunca mistura — cada JSON é único para cada marca. Cinco clientes, cinco estilos completamente distintos e consistentes.

3. Um professor de design chamado Henrique quer ensinar seus alunos a analisar o design de grandes marcas. Em vez de mostrar slides com explicações abstratas sobre "harmonia de cores" e "hierarquia tipográfica", ele usa o Site-to-JSON para extrair os dados reais de sites famosos. Os alunos veem os números concretos: "a Apple usa Helvetica Neue em 17px para o corpo do texto e 56px para o título principal, com um espaçamento de 160% entre as linhas". De abstrato e teórico, o aprendizado se torna concreto e verificável. Os alunos entendem de verdade o que antes era só intuição.
