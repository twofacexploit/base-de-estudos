## O que é

Um **design system** é um documento que define a identidade visual completa de um produto: cores, fontes, espaçamentos, componentes, motion, layout. Tradicionalmente, ficava em ferramentas como Figma ou Zeroheight.

**DESIGN.md** é a ideia de que um design system **pode morar num arquivo Markdown** — que **qualquer agente de IA entende e aplica automaticamente**.

## A plataforma

**getdesign.md** — galeria de DESIGN.mds inspirados em sites reais. Você pode clonar o design de referência com um comando.

## Instalação

```bash
npx get-design stripe    # instala design inspirado na Stripe
npx get-design linear    # instala design inspirado no Linear
npx get-design vercel    # instala design inspirado na Vercel
```

Um comando → DESIGN.md completo no seu projeto → todo código novo que você gerar já respeita essa identidade.

## O que um DESIGN.md contém

```markdown
# DESIGN.md

## Visual Identity
- Cores primárias, secundárias, backgrounds
- Paleta de apoio (success, warning, error)

## Typography
- Fontes (famílias)
- Escala de tamanhos (display, title, body, caption)
- Pesos (400, 500, 600, 700)

## Spacing System
- Unidade base (4px, 8px)
- Ritmo: 4, 8, 16, 24, 32, 48, 64...

## Components
- Buttons (primary, secondary, ghost)
- Cards (default, elevated, outlined)
- Inputs (default, error, disabled)

## Motion
- Transições padrão
- Curves (ease-in, ease-out)
- Durações

## Layout
- Grid (12 colunas, 16px gutter)
- Breakpoints (sm, md, lg, xl)
- Max-widths por contexto
```

## O sistema completo de arquivos

DESIGN.md faz parte de um sistema maior que inclui outros arquivos de contexto para agentes:

```
CLAUDE.md   → comportamento do agente
DESIGN.md   → identidade visual
SOUL.md     → personalidade
USER.md     → perfil do usuário
MEMORY.md   → histórico e decisões
TOOLS.md    → ferramentas disponíveis
```

Cada arquivo define **uma dimensão** do agente. Juntos, formam um "personagem" consistente que sabe tudo sobre o contexto do projeto.

## Por que Markdown em vez de JSON?

- **Legibilidade** — você lê o arquivo e entende
- **Edição manual** — qualquer pessoa atualiza
- **LLMs interpretam perfeitamente** — markdown é a linguagem nativa deles
- **Versionamento simples** — é texto, git funciona bem

JSON é ótimo pra máquinas, mas Markdown é ótimo pra **humanos e LLMs simultaneamente**.
