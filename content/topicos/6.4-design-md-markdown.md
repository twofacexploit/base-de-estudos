## O que é

Um **design system** é como o manual de identidade visual de uma marca — define as cores oficiais, os tipos de letra, o espaçamento entre elementos, como os botões devem parecer. Antes, esse manual ficava em ferramentas caras como o Figma.

**DESIGN.md** é a ideia de guardar esse manual num simples arquivo de texto — que qualquer assistente de IA lê e segue automaticamente, sem que você precise explicar nada.

## A plataforma

**getdesign.md** — uma galeria de arquivos DESIGN.md inspirados em grandes sites. Você instala o estilo de um site famoso com um único comando.

## Instalação

```bash
npx get-design stripe    # instala design inspirado na Stripe
npx get-design linear    # instala design inspirado no Linear
npx get-design vercel    # instala design inspirado na Vercel
```

Um comando e pronto — o DESIGN.md entra no seu projeto e todo código novo criado pela IA já respeita aquele estilo visual automaticamente.

## O que um DESIGN.md contém

```markdown
# DESIGN.md

## Visual Identity
- Cores primárias, secundárias, backgrounds
- Paleta de apoio (success, warning, error)

## Typography
- Fontes (famílias)
- Escala de tamanhos (display, title, body, caption)
- Pesos (400, 500, 600, 700)

## Spacing System
- Unidade base (4px, 8px)
- Ritmo: 4, 8, 16, 24, 32, 48, 64...

## Components
- Buttons (primary, secondary, ghost)
- Cards (default, elevated, outlined)
- Inputs (default, error, disabled)

## Motion
- Transições padrão
- Curves (ease-in, ease-out)
- Durações

## Layout
- Grid (12 colunas, 16px gutter)
- Breakpoints (sm, md, lg, xl)
- Max-widths por contexto
```

## Adoção explosiva

**35.000 estrelas em 1 semana** na plataforma onde desenvolvedores compartilham projetos — isso significa que muita gente estava com o mesmo problema: como fazer a IA criar coisas visualmente consistentes. O DESIGN.md resolve isso.

## O sistema completo de arquivos

O DESIGN.md faz parte de um grupo de arquivos que, juntos, ensinam uma IA tudo sobre um projeto:

```
CLAUDE.md   → como o assistente deve se comportar
DESIGN.md   → como as coisas devem parecer visualmente
SOUL.md     → a personalidade do produto
USER.md     → quem são os usuários
MEMORY.md   → histórico e decisões já tomadas
TOOLS.md    → ferramentas disponíveis
```

Cada arquivo define uma parte do "personagem" do projeto. Juntos, a IA entende o contexto completo e trabalha com muito mais precisão.

## Por que Markdown em vez de outros formatos?

- **Qualquer pessoa consegue ler** — não precisa saber programar
- **Fácil de editar** — você altera como um documento de texto comum
- **IAs entendem perfeitamente** — Markdown é quase a língua nativa dos assistentes de IA
- **Fácil de versionar** — é texto puro, sem formatos proprietários

É o equilíbrio perfeito: legível para humanos e para máquinas ao mesmo tempo.

## Exemplos do dia a dia

1. **Criando um site com identidade visual consistente:** você quer que seu site pareça com o da Stripe — aquele visual clean, profissional e moderno. Em vez de tentar descrever isso com palavras, você roda um comando que instala o DESIGN.md da Stripe no seu projeto. A partir daí, toda vez que pede para a IA criar uma tela, ela já segue aquele estilo automaticamente.

2. **Trabalhando em equipe com consistência:** uma equipe de 3 pessoas está criando um app juntas. Sem um guia de design, cada pessoa cria botões e cores diferentes — fica uma bagunça. Com o DESIGN.md, todas as três seguem o mesmo arquivo de referência, e a IA de cada uma gera código visualmente idêntico, sem precisar combinar nada verbalmente.
