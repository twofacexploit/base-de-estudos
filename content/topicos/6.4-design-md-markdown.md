## O que é um design system

Um design system é o manual de identidade visual de um produto digital — ele define as cores oficiais que podem ser usadas, os tipos de letra aprovados, o tamanho dos espaços entre elementos, como os botões devem aparecer, como os campos de formulário se comportam. É como o manual da marca de uma empresa grande, mas com o nível de detalhe técnico necessário para que programadores e designers façam tudo de forma consistente.

Sem esse manual, cada pessoa que trabalha no projeto toma suas próprias decisões visuais. Um programador usa um botão azul claro, outro usa azul escuro. Uma página tem bastante espaço em branco, outra parece apertada. O resultado é um produto que parece montado por partes que não combinam — o que passa uma impressão de amadorismo mesmo quando o conteúdo é excelente.

Grandes empresas como Google, Apple, Airbnb e Spotify têm design systems complexos que garantem que cada tela de cada produto pareça parte da mesma família. O Material Design do Google é um exemplo famoso — é por isso que todos os aplicativos do Google têm um visual consistente, independente de qual deles você está usando.

## O que é o DESIGN.md

O DESIGN.md é uma ideia simples e poderosa: em vez de guardar esse manual de identidade visual em ferramentas caras e complexas como o Figma, você o escreve num simples arquivo de texto chamado DESIGN.md. Esse arquivo usa uma linguagem chamada Markdown — uma forma de escrever texto com formatação básica, como negrito e títulos, sem precisar de nenhum programa especial.

A grande vantagem é que qualquer assistente de IA lê e entende um arquivo de texto perfeitamente. Quando o DESIGN.md está no projeto, o Claude ou qualquer outro assistente sabe automaticamente as cores aprovadas, as fontes corretas e como os elementos devem se comportar — e aplica tudo isso em cada linha de código que cria, sem precisar que você explique nada.

Markdown é a linguagem que você usa quando escreve um texto com \*\*negrito\*\* ou um # para criar um título. É simples ao ponto de qualquer pessoa conseguir ler — mesmo sem saber programar.

## A plataforma getdesign.md

O `getdesign.md` é um site que reúne arquivos DESIGN.md inspirados nos melhores e mais respeitados produtos digitais do mundo. Você instala o estilo de um site famoso com um único comando digitado no computador — e em segundos o seu projeto tem um guia visual completo baseado naquele referencial.

O comando `npx get-design stripe` instala um DESIGN.md inspirado no design da Stripe — aquela empresa americana de pagamentos conhecida por ter um dos visuais mais refinados e profissionais da indústria de tecnologia. O comando `npx get-design linear` instala um design inspirado no Linear, ferramenta de gestão de tarefas com visual extremamente limpo e moderno. O `npx get-design vercel` instala um design baseado na identidade visual da Vercel, conhecida pelo minimalismo em preto e branco.

"npx" é uma forma de rodar um comando no computador diretamente, sem precisar instalar nada antes — como apertar um botão que executa uma ação automaticamente.

## O que um DESIGN.md contém

Um arquivo DESIGN.md bem feito cobre todas as dimensões visuais de um produto. A seção de identidade visual traz as cores primárias e secundárias com os códigos exatos — não "azul", mas "#1A73E8" — além das cores de fundo, de destaque, de sucesso, de aviso e de erro.

A seção de tipografia define as famílias de fontes aprovadas, a escala de tamanhos para cada tipo de texto (título principal grande, subtítulo, texto normal, texto pequeno de rodapé), e os pesos de cada fonte (normal, médio, negrito, extranegrito).

O sistema de espaçamento define a unidade base e a progressão — por exemplo, os espaçamentos usam múltiplos de 8 pixels: 8, 16, 24, 32, 48, 64. Isso cria harmonia visual sem que ninguém precise pensar em espaçamento a cada novo elemento.

A seção de componentes descreve os botões (primário, secundário, fantasma), os cartões (padrão, elevado, com borda), os campos de formulário (normal, com erro, desabilitado). A seção de movimento define as transições padrão — quanto tempo leva uma animação, qual curva ela segue. E o layout define quantas colunas a página tem, o espaço entre elas, e em quais tamanhos de tela a página se reorganiza para funcionar bem no celular.

## A adesão explosiva da ideia

O conceito de DESIGN.md acumulou 35.000 estrelas em uma semana na plataforma onde desenvolvedores compartilham projetos — o GitHub. Isso é um sinal claro de que muita gente estava com o mesmo problema: como fazer um assistente de IA criar coisas visualmente consistentes sem precisar corrigir a aparência a todo momento. O DESIGN.md resolve exatamente isso.

## O sistema completo de arquivos

O DESIGN.md faz parte de um grupo de arquivos especiais que, juntos, criam um contexto completo para qualquer assistente de IA que trabalha num projeto. Cada arquivo ensina uma dimensão diferente do que é o projeto.

O CLAUDE.md define como o assistente deve se comportar — qual tom usar, quais decisões tomar sozinho e quais consultar o dono do projeto. O DESIGN.md define como as coisas devem parecer visualmente. O SOUL.md descreve a personalidade do produto — se é sério e corporativo, casual e amigável, ou técnico e preciso. O USER.md define quem são os usuários — idade, nível de conhecimento técnico, necessidades e objetivos. O MEMORY.md guarda o histórico de decisões já tomadas no projeto — para que o assistente não sugira algo que já foi decidido de outra forma. E o TOOLS.md lista as ferramentas disponíveis para o assistente usar.

Juntos, esses arquivos dão ao assistente de IA uma visão completa do projeto — como um briefing detalhado que um novo funcionário receberia no primeiro dia de trabalho.

## Por que Markdown e não outra coisa

Markdown tem uma combinação de qualidades difíceis de encontrar em outros formatos. Qualquer pessoa consegue ler sem precisar de nenhum programa — é texto puro, como um e-mail. É fácil de editar com qualquer editor de texto. Os assistentes de IA entendem perfeitamente, porque grande parte da web é escrita em Markdown — é quase a língua nativa dessas ferramentas. E é fácil de versionar, ou seja, de guardar um histórico de alterações — porque é texto simples, sem formatos proprietários que precisem de um programa específico para abrir.

## Por que isso importa para você?

Consistência visual é o que separa produtos que parecem profissionais dos que parecem amadores — mesmo quando o conteúdo de ambos é igualmente bom. Um arquivo DESIGN.md bem feito garante essa consistência automaticamente, sem que cada pessoa do time precise tomar decisões visuais do zero a cada nova tela.

## Exemplos do dia a dia

1. **Criando um site com visual de empresa grande sem ter designer:** você quer que o seu site pareça com o da Stripe — aquele visual clean, profissional e sem excesso, que passa seriedade e confiança imediata. Em vez de tentar reproduzir esse estilo na base do olhômetro, você roda o comando `npx get-design stripe` no computador. O arquivo DESIGN.md entra no projeto. A partir daí, toda vez que você pede para o Claude criar uma nova página ou um novo componente, ele automaticamente usa as cores, fontes e espaçamentos definidos no arquivo. O resultado é um site visualmente consistente e profissional, mesmo que você nunca tenha estudado design.

2. **Equipe que cria sem precisar combinar tudo verbalmente:** três pessoas estão construindo um aplicativo juntas, cada uma responsável por partes diferentes. Sem um guia de design, cada uma tomaria decisões visuais diferentes: uma usaria botões azuis, outra verdes, outra com bordas arredondadas diferentes. O resultado seria um aplicativo inconsistente que parece feito por pessoas que nunca conversaram. Com o DESIGN.md, as três seguem o mesmo arquivo automaticamente — e o assistente de IA de cada uma gera código que parece ter saído da mesma mão, sem nenhuma reunião necessária para alinhar o visual.

3. **Renovando a aparência de um site sem perder consistência:** você tem um site antigo e quer modernizar o visual. Em vez de mudar cores e fontes aleatoriamente — correndo o risco de que alguns elementos fiquem com a cor nova e outros com a antiga — você começa criando o DESIGN.md com o novo visual definido. A partir daí, cada elemento que for atualizado segue automaticamente as novas especificações. Quando o trabalho terminar, o site todo terá o mesmo visual novo, sem inconsistências, porque tudo foi guiado pelo mesmo arquivo de referência.
