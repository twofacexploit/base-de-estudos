## O problema

LLMs têm uma janela de contexto limitada. Claude **esquece tudo** entre sessões — você fecha o Claude Code, reabre amanhã, e ele não lembra de nada do projeto. Cada conversa começa do zero.

Isso gera:
- **Desperdício de tempo** reexplicando o mesmo contexto
- **Erros repetidos** porque "esqueceu" de uma decisão anterior
- **Alto consumo de tokens** mandando tudo de novo

## A solução

**Claude-Mem** é um plugin que:

1. No fim da sessão, **comprime** tudo que foi dito em um resumo estruturado
2. Salva num arquivo local (`.claude-mem/`)
3. Na próxima sessão, carrega esse resumo automaticamente no contexto

Pronto — Claude "lembra" do projeto.

## Resultados reais

- **95% menos tokens** por sessão (só carrega o resumo, não a conversa inteira)
- **20x mais tool calls** antes de esbarrar no limite de contexto
- **100% open source** e gratuito

## A analogia

É a mesma lógica do cérebro humano: você não "lembra" cada palavra de uma conversa de ontem — lembra do **resumo, das decisões, do que foi importante**. Claude-Mem faz isso explicitamente.

## Como começa

```bash
# instalar o plugin no Claude Code
# configurar a pasta de memória do projeto
# pronto — cada sessão já carrega o contexto anterior
```

## Cuidado

Resumos podem perder nuance. Se uma decisão importante foi tomada "no meio da conversa", precisa garantir que o resumo capturou. Por isso, projetos grandes ainda se beneficiam de ter **MEMORY.md** explícito como complemento.
