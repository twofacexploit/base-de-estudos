## O que é isso, em linguagem simples

Imagine que você comprou um aparelho de televisão novo. Antigamente, você precisava entrar nos menus, configurar o sinal de antena, ajustar o contraste, configurar o idioma, vincular cada aplicativo separadamente. Levava horas. Hoje, você liga o aparelho, ele detecta a sua rede Wi-Fi, pergunta sua localização, e se configura praticamente sozinho em minutos.

O **AutoSkills** faz algo parecido para projetos que usam inteligência artificial. Quando você começa um projeto novo, normalmente precisaria pesquisar quais habilidades de IA são relevantes para o que você está construindo, baixar cada uma manualmente, configurar onde cada uma deve ficar, testar se estão funcionando. Dependendo da complexidade do projeto, poderia levar horas.

Com o AutoSkills, você roda um único comando. Ele olha para o seu projeto — como um técnico experiente que examina a situação — detecta automaticamente quais tecnologias você está usando, e instala as habilidades de IA corretas para aquelas tecnologias específicas. Tudo configurado no lugar certo, pronto para usar.

## Por que isso importa para você?

Quando alguém trabalha com tecnologia, cada projeto usa diferentes ferramentas — é como dizer que cada prato de um restaurante usa ingredientes diferentes. Um bolo usa ovos, farinha e açúcar. Uma sopa usa legumes e caldo. Não faz sentido ter todos os ingredientes da cozinha preparados para cada prato — só os que aquele prato específico precisa.

Da mesma forma, um projeto que usa certas tecnologias precisa de habilidades de IA específicas para aquelas tecnologias. Se você usa Shopify para sua loja, precisa das habilidades de IA que entendem como o Shopify funciona. Se usa Stripe para pagamentos, precisa das habilidades que entendem o Stripe. Não faz sentido ter habilidades de IA para tecnologias que você não usa — só ocupam espaço e causam confusão.

O AutoSkills detecta o que você tem, instala o que faz sentido, e ignora o resto. Inteligente e eficiente.

## Como usar — é muito simples

```bash
npx autoskills             # detecta e instala tudo automaticamente
npx autoskills --dry-run   # mostra uma prévia sem instalar nada
```

É como dois modos de um aspirador de pó: o normal que limpa de verdade, e o modo de visualização que mostra onde vai passar antes de começar. Você decide se quer ver o plano antes ou ir direto para a ação.

## O que ele faz — os 4 passos por baixo dos panos

**Passo 1 — Leitura:** A ferramenta lê os arquivos de configuração do seu projeto. Esses arquivos são como a lista de ingredientes de uma receita — eles listam tudo que o projeto usa. A ferramenta não precisa abrir o projeto inteiro; ela lê só os arquivos que declaram as dependências.

**Passo 2 — Identificação:** Com base no que leu, a ferramenta identifica quais tecnologias estão presentes. É como um mecânico que abre o capô do carro e, em segundos, sabe que modelo é, qual motor tem, quais são os sistemas instalados. Sem precisar testar nada — só de ver.

**Passo 3 — Busca:** A ferramenta consulta uma biblioteca de habilidades de IA e encontra as que são compatíveis e úteis para aquelas tecnologias específicas. É como um sommelier que, sabendo que você gosta de carne vermelha, já sabe exatamente qual vinho recomendar — sem precisar te mostrar toda a carta.

**Passo 4 — Instalação:** As habilidades são baixadas e colocadas automaticamente na pasta correta do projeto. Você não precisa saber onde colocar cada coisa. A ferramenta conhece a estrutura e organiza por você.

## Mais de 50 tecnologias reconhecidas

A ferramenta reconhece praticamente qualquer combinação de tecnologias modernas. Alguns exemplos que você pode ter ouvido falar:

- **Lojas e pagamentos:** Shopify, Stripe
- **Bancos de dados:** Supabase, Prisma, Neon
- **Serviços de nuvem:** AWS, Cloudflare, Vercel
- **Frameworks de site:** React, Next.js, Vue, Svelte, Astro
- **Estilos visuais:** Tailwind
- **Aplicativos de celular:** Expo (React Native)
- **Testes:** Playwright
- **Animações:** GSAP

Se você usa combinações como "Shopify + Stripe + React" ou "Next.js + Supabase + Tailwind", a ferramenta identifica cada uma e instala as habilidades certas para cada tecnologia presente.

## O modo --dry-run — ver antes de fazer

O `--dry-run` (que significa "ensaio sem efeito real") mostra exatamente o que seria instalado, sem instalar nada de verdade. É como fazer a lista do supermercado antes de ir — você vê o total, decide se faz sentido, e só então vai comprar.

Este modo é útil em três situações:

**Quando você quer revisar antes de confirmar:** Ver o que vai ser adicionado ao projeto antes de tomar a decisão. Especialmente importante em projetos que outras pessoas também usam.

**Quando você quer descobrir habilidades novas:** Às vezes a ferramenta identifica habilidades que você não conhecia para tecnologias que você usa. É uma forma de descobrir recursos que você não sabia que existiam.

**Quando você quer evitar surpresas:** Em projetos mais sensíveis, nada melhor do que saber exatamente o que vai acontecer antes de apertar o botão de confirmar.

## A filosofia por trás — automação inteligente, não automação cega

Existe uma diferença importante entre uma ferramenta que instala tudo que existe e uma que instala só o que faz sentido para você. A segunda é muito mais útil.

O AutoSkills não é um catálogo que joga tudo em cima. É um diagnóstico que entende o contexto e recomenda o que é relevante. Como a diferença entre um médico que escuta seus sintomas e prescreve o remédio certo, versus uma farmácia que vende tudo sem perguntar.

## Exemplos do dia a dia

1. Uma costureira chamada Renata vendeu tanto que contratou um desenvolvedor para criar sua loja online. O desenvolvedor escolheu usar Shopify para a loja e Stripe para os pagamentos — tecnologias consolidadas e confiáveis. Quando rodou o AutoSkills, a ferramenta detectou automaticamente que o projeto usava Shopify e Stripe e instalou as habilidades de IA específicas para e-commerce e pagamentos. O desenvolvedor não precisou pesquisar quais habilidades eram relevantes — a ferramenta já conhecia o cenário e instalou o que fazia sentido. Duas horas de pesquisa e configuração manual se tornaram dois minutos de automação.

2. Um estudante de tecnologia chamado Gabriel começou a criar um aplicativo de cardápio digital para seu trabalho de conclusão de curso. Usou React para a parte visual e Firebase para guardar os dados. Quando rodou o AutoSkills pela primeira vez, a ferramenta detectou essas tecnologias e instalou habilidades de IA específicas para React e Firebase. Gabriel ficou surpreso ao ver que existiam habilidades prontas para exatamente o que ele estava construindo — ele nem sabia que poderia ter ajuda tão específica. Em vez de perder um fim de semana descobrindo como as ferramentas se encaixavam, ele passou o fim de semana desenvolvendo o aplicativo em si.

3. Uma agência de marketing digital chamada Impulso tem uma equipe de 6 pessoas que começa projetos novos para clientes toda semana. Cada projeto usa uma combinação diferente de tecnologias — cada cliente tem preferências e sistemas diferentes. Antes do AutoSkills, um desenvolvedor passava de 2 a 3 horas por projeto novo configurando as habilidades de IA corretas. Com o AutoSkills, essa etapa caiu para menos de 5 minutos. Em uma equipe que inicia 3 projetos por semana, a economia é de mais de 18 horas mensais — tempo que volta para o trabalho criativo que realmente diferencia a agência.
