## A pirâmide (da base ao topo)

```
🔝 APIs (REST + JSON, GraphQL)
🔸 Architectural Patterns (Monolithic, Microservices, Serverless)
🟤 Caching (CDN, Redis, Memcached)
🟠 CI/CD (GitHub, GitLab, Jenkins, CircleCI)
🟡 Deployment (AWS, Azure, GCP, Docker, Kubernetes, Vercel)
🔴 AI Integration (GPT, Claude, Gemini, DeepSeek, Mistral, LLaMA)
🟢 Frameworks (Spring Boot, Node.js, Django, FastAPI, LangChain)
🔵 Databases (PostgreSQL, MySQL, MongoDB, DynamoDB, Cassandra)
🟣 Programming Languages (Java, Python, C#, Ruby, PHP, JS, Go, Rust)
```

Cada camada depende da anterior. **Linguagem** é a base — escolhida a linguagem, o **framework** segue. Framework escolhido, **banco** entra. E assim por diante.

## REST vs GraphQL — a diferença essencial

| | REST | GraphQL |
|---|---|---|
| **Como funciona** | Endpoints fixos, retornam tudo sempre | Cliente pede só o que precisa |
| **Eficiência** | Overfetching comum | Resposta exata, menor |
| **Aprendizado** | Fácil | Curva maior |
| **Quando usar** | APIs simples, públicas | Apps complexas, mobile |

**REST:** endpoints como `/users`, `/posts`. Retorna **TUDO** — todas as propriedades do usuário, todos os campos. Muito dado desperdiçado.

**GraphQL:** você pergunta: `{ user { name, email } }` — recebe **só** name e email. Mais eficiente, principalmente em mobile com conexão ruim.

## Serverless — o que significa

**Serverless** = "sem servidor" — mas não é bem isso. Significa que **você não gerencia o servidor**. Você escreve funções, e a cloud decide onde e quando executá-las.

**Vantagens:**
- **Paga só pelo uso** (execuções, não por servidor rodando)
- **Escala automática** — 1 ou 1 milhão de requests, funciona
- **Zero manutenção** de OS, patches, etc

**Desvantagens:**
- **Cold start** — primeira request demora mais
- **Limitado em execução longa** (timeouts)
- **Vendor lock-in** (código diferente por provider)

**Exemplos:** AWS Lambda, Vercel Functions, Cloudflare Workers.

## Como essas camadas viram um produto real

Um exemplo concreto — um e-commerce simples:

```
Linguagem:    TypeScript  
Framework:    Next.js  
Banco:        PostgreSQL (Supabase)  
Cache:        Redis (Upstash)  
API:          REST + alguma coisa GraphQL  
IA:           Claude para recomendações  
Deploy:       Vercel (serverless)  
CI/CD:        GitHub Actions  
```

Tudo que aparece na pirâmide está presente. É assim que produtos modernos são construídos.

## A armadilha

Muita gente tenta **aprender tudo** antes de construir. Errado. **Escolha a menor stack possível que resolve seu problema**, construa, lance, aprenda o resto depois. Quem espera dominar tudo nunca lança nada.
