## O que é

**DefenseClaw** (Cisco) é um sistema que **monitora e bloqueia ações perigosas de agentes de IA em tempo real**. É como um filtro entre o agente e o mundo real.

## O problema que resolve

Agentes de IA ganham cada vez mais "poder": podem executar comandos, editar arquivos, fazer requisições pra APIs. O problema é que um prompt malicioso ou uma alucinação pode levar o agente a fazer algo catastrófico — deletar dados, vazar informação, gastar dinheiro.

## Classificação de ações

O DefenseClaw categoriza tudo que o agente tenta fazer em três níveis:

```
SEGURO:    ler arquivo, buscar documentação, consultar API pública
SUSPEITO:  acessar arquivo fora do projeto, escrever em pasta nova
PERIGOSO:  deletar em massa, enviar dados pra servidor externo,
           executar comandos privilegiados
```

Ações **perigosas** são bloqueadas por padrão — o humano precisa aprovar uma a uma.

## Especialmente importante em

- **Ambientes corporativos** onde um erro custa caro
- **Infraestrutura crítica** (hospitais, energia, financeiro)
- **Produção** (não é o mesmo que ambiente de desenvolvimento)

## A analogia

É o mesmo conceito dos modos de **permissão** do Claude Code: o agente não age por conta própria em ações destrutivas. O DefenseClaw traz essa filosofia pra **qualquer agente**, de qualquer fornecedor.
