## Vibe Coder caótico (como NÃO fazer)

```
Copia → Cola → Reza → ERROR → Repete
```

Isso é o que a maioria faz. Abre o ChatGPT, pede código, cola, dá erro, volta, pede de novo, cola, roda, erra. Nunca para pra pensar no **sistema** — só reage.

## Vibe Coding real (com método)

```
1. Configura ambiente (Claude Code, preferências, regras)
2. Transforma IA em colaborador (skills, MCPs, testa agente)
3. Planeja ANTES de codar (PRD, quebra em tarefas)
4. Constrói com método (MVP 70%, itera, debug com contexto)
```

A diferença: **você constrói um sistema** onde a IA vira parte do processo — não um ajudante eventual.

## A frase que resume tudo

> *"Beginners prompt. Experienced builders design systems."*

Traduzido: iniciantes ficam escrevendo prompts melhores. Experientes **desenham o sistema** onde prompts nem são mais a parte principal — regras, skills, hooks e workflows fazem o trabalho.

## MVP 70%

Lance com **70% funcionando**. Colete feedback real. Melhore em cima de dados reais. Não espere 100% — ninguém nunca vai chegar lá sozinho, e o mercado muda antes de você terminar.

## A transformação mental

Quem entende isso para de **usar** IA e começa a **operar** IA — como se fosse infraestrutura. É a diferença entre "peguei um táxi" e "tenho uma frota rodando pra mim 24h".
