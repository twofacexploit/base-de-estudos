## A regra de ouro

Linguagens de programação são como idiomas: não existe "o melhor idioma do mundo". Existe o idioma certo para o lugar onde você vai. Japonês é perfeito para o Japão, mas inútil se você quer trabalhar na França. O mesmo vale para programação — a linguagem certa depende do que você quer construir. Quem fica pulando de linguagem em linguagem sem foco não aprende nenhuma direito.

## Web Development

As linguagens que constroem sites e aplicações que você usa no navegador:

**HTML, CSS, JavaScript, TypeScript, PHP, React JS, Angular, Python, Java, SQL**

HTML e CSS são a estrutura e a pintura de um site — todo site do mundo os usa. JavaScript faz as coisas se moverem e reagirem. React é a ferramenta mais popular hoje para construir interfaces modernas.

## Software Development

Para criar programas que rodam no computador, aplicativos de celular e sistemas internos:

**C, C++, Java, Python, C#, Swift, Kotlin, Rust, Go, Ruby**

C e C++ ainda dominam onde velocidade é crítica — jogos e sistemas operacionais. Swift é para criar apps no iPhone. Kotlin é para criar apps no Android. Rust é considerado o futuro em muitas áreas por ser muito seguro e rápido.

## Machine Learning

A área que ensina computadores a aprender e tomar decisões sozinhos:

**Python (rainha), R, Julia, Java, C++, Scala, MATLAB, JavaScript, Lisp, Prolog**

Python domina porque tem as "caixas de ferramentas" mais completas para IA. R é forte em pesquisa científica e estatística. Julia é uma linguagem nova que está crescendo no meio acadêmico.

## O destaque: Python

**Python é a linguagem mais versátil hoje.** Ela aparece nas três categorias acima — sites, software e IA. Se você quer uma única linguagem que funciona para muita coisa, Python é a resposta.

## Por onde começar?

Depende do que você quer fazer:

### Quer criar sites?
```
HTML → CSS → JavaScript → React
```
Cada etapa constrói em cima da anterior. Sequência clássica e comprovada.

### Quer trabalhar com IA?
```
Python → NumPy → Pandas → TensorFlow/PyTorch
```
Python é o ponto de entrada obrigatório. As bibliotecas (conjuntos de ferramentas) vêm depois.

### Quer cibersegurança?
```
Python + C + Bash
```
Python para automatizar tarefas, C para entender como ataques funcionam, Bash para criar scripts no Linux.

### Quer apps mobile?
- **Android** → Kotlin
- **iOS** → Swift
- **Ambos ao mesmo tempo** → React Native ou Flutter

## A pior pergunta

**"Qual linguagem devo aprender primeiro?"**

Essa pergunta não tem resposta sem saber o que você quer fazer. É como perguntar "qual curso fazer?" sem saber qual profissão quer seguir. A pergunta certa é: **"Qual problema eu quero resolver?"** — daí a linguagem surge naturalmente.

## Exemplos do dia a dia

1. **Decidindo por onde começar:** você quer criar uma loja virtual para vender artesanato. Isso é um site — então o caminho é HTML, depois CSS, depois JavaScript. Você não precisa aprender Python nem Kotlin. Escolher a linguagem certa desde o começo economiza meses de estudo desnecessário.

2. **Mudando de área:** você é contador e quer automatizar relatórios que faz no Excel toda semana. Para isso, Python é a escolha perfeita — é fácil de aprender e já tem ferramentas prontas para trabalhar com planilhas. Nesse caso, aprender a criar sites não ajudaria em nada.
